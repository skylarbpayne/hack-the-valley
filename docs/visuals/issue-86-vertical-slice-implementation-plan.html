Hack the Valley issue #86 · vertical implementation plan

Stop building layers. Ship usable planning workflows.

The planning timeline is still the domain spine, but implementation now moves in vertical slices: each phase includes the schema, domain command, admin API, admin UI, tests, screenshots, and acceptance path for one real organizer workflow.

Blue = end-to-end feature sliceGreen = validated user workflowOrange = approval gateRose = anti-pattern / stop signViolet = reusable mechanism proven by a slice
Old planSchema → API → UI → lanes. Clean-looking, slow feedback, easy to overbuild.
New planCycle MVP → checks/reminders → meetings → sponsor/print → readiness → campaign drafts → handoff.
Quality barEvery slice must be usable in /admin, tested, screenshot, and safe from accidental external side effects.

Vertical slice roadmap

Definition of done per slice

  • Real admin workflow available in /admin.
  • Minimum schema/domain/API/UI required for that workflow only.
  • Domain and route tests pass.
  • Browser screenshot evidence in PR body.
  • No external send, publish, purchase, or production migration without approval.

Forbidden shortcut

  • “Let’s create all tables first.” No.
  • “The API works, UI later.” Also no.
  • “Metadata can hold everything forever.” That’s a junk drawer with JSON perfume.
  • “Tests pass, no screenshot needed.” UI work needs pixels.
