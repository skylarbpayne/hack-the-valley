Hack the Valley issue #86 · Planning / management system

Make the timeline the spine, not another admin junk drawer.

The flexible object is a typed PlanningItem: every milestone, checklist item, meeting, sponsor follow-up, print order, volunteer checkpoint, social post, and email blast becomes one item on the HTV 2027 planning timeline. New checks/items are added by creating templates or item types, not by adding one-off columns/routes forever.

Recommendation: after this planning artifact is approved, the first implementation PR should be the core spine only: planning vocabulary, migrations, read/write operations with tests, and a minimal admin timeline view. Do not build sponsorship, social, print, and volunteer “mini apps” first. That’s how this turns into mud with a login screen.
